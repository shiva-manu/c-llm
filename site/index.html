Switch providers in one line

Write once against the unified API. Change a string to switch providers.

Basic Streaming Fallback 
LLMRequest req = {
    .model       = "gpt-4o-mini",
    .prompt      = "Explain monads in simple terms.",
    .system      = "You are a patient teacher.",
    .max_tokens  = 200,
    .temperature = 0.7f
};

// Use OpenAI
LLMResponse res = llm_generate("openai", &req);

// Swap to Claude -- that's it
LLMResponse res = llm_generate("claude", &req);

// Or use Groq for speed
LLMResponse res = llm_generate("groq", &req);
void on_token(const char *token, void *ud) {
    printf("%s", token);
    fflush(stdout);
}

LLMRequest req = {
    .model      = "gpt-4o-mini",
    .prompt     = "Write a short story.",
    .max_tokens = 500,
    .stream     = 1,
    .stream_cb  = on_token,
};

LLMResponse res = llm_generate_stream("openai", &req);
const char *providers[] = {"groq", "openai", "claude", NULL};
LLMResponse res = {0};

for (int i = 0; providers[i]; i++) {
    res = llm_generate(providers[i], &req);
    if (res.success) break;
    fprintf(stderr, "%s failed: %s\n", providers[i], res.error);
}

Built for Real Applications

Everything you need to ship LLM-powered C and C++ software.

⚡

Streaming (SSE)

Real-time token delivery via callback. Handles OpenAI, Claude, and Gemini SSE formats internally.

🔍

Tool / Function Calling

Define tools with JSON schemas. Unified parsing of tool calls from OpenAI, Claude, and Gemini.

🖼

Multimodal / Vision

Send image URLs or base64 data URIs as input. Claude, OpenAI, and Gemini vision support built in.

💬

Multi-turn Conversations

Pass full conversation history with system, user, and assistant roles. Works across all providers.

📊

Embeddings

Generate vector embeddings via OpenAI, Cohere, or Gemini with the companion llm_embed() function.

🔧

JSON Mode

Request structured JSON output. Each provider's native format is handled transparently.

🛠

Retry with Backoff

Automatic retries on 429/5xx errors with exponential backoff and Retry-After header support.

🔒

Thread-Safe

No global state. All state is local or passed through function arguments. Safe for concurrent use.

📦

Static Library

Link libllm.a into your project. No runtime dependencies, no dynamic linking headaches.

Clean Architecture

A dispatch table routes calls to provider-specific implementations. Shared code minimizes duplication.

Get Started in 3 Steps

From zero to LLM-powered C code in under a minute.

1

Install Dependencies

sudo apt install libcurl4-openssl-dev libcjson-dev
2

Build the Library

git clone https://github.com/shivamani/anylm.git
cd anylm
cmake -S . -B build && cmake --build build -j
3

Link and Use

gcc -std=c99 -Iinclude -o myapp myapp.c \
  -Lbuild -lllm -lcurl -lcjson

Stop rewriting integrations.

AnyLM gives you one function call for 17 providers. Ship faster, switch freely.
